# Cyber Xiuxian / 赛博修仙

> 一个基于中文语境、东方符箓与修仙想象的开源世界观协议，用来重新叙事 AIGC、Agent、区块链、自动化工作流与未来智能网络。

## 这是什么

**赛博修仙**不是把技术神秘化，也不是简单给英文术语换一层古风皮。

它试图建立一套可操作、可泛化、可共建的概念体系：用中文文化中的符、咒、箓、印、坛、阵、灵机、天道等结构，解释现代人工智能与赛博技术的运行方式。

在这个体系里：

- 写 prompt 像画符。
- 调用模型像起咒。
- token 是符。
- 电力是灵气。
- 算力是灵力。
- 数据是灵材。
- 模型是灵机。
- 模型核心是灵核。
- workflow 是阵法。
- agent 是分身。
- 区块链是天账。
- 共识机制是天道的一种显化。
- 调模型与调参实验，是炼丹。

我们希望它既能被创作者拿来讲故事，也能被开发者拿来解释技术，还能被 geek 社区继续 fork、扩展和争论。

## 核心命题

现代智能系统越来越复杂，普通用户很难直接理解其中的抽象机制。与此同时，中文世界长期依赖英文术语、直译术语和工程黑话来描述 AI。

**赛博修仙**的基本判断是：

> 技术需要精确，也需要叙事。一个好的概念系统，应该既能解释机制，也能激发想象。

因此，本项目不是要取代现有技术术语，而是提供一套新的中文解释层。

它可以作为：

- AIGC 科普语言
- AI 产品的世界观设定
- prompt engineering 教学隐喻
- agent workflow 的可视化叙事
- Web3 / 区块链机制的东方化解释
- 科幻、游戏、小说、视频内容的设定基础
- 一个开放的中文技术文化实验

## 基本语法

传统符咒体系可以被抽象为一套操作语法：

> 身份授权 + 指令书写 + 语音激活 + 权威印证 + 仪式流程 + 执行对象 + 结果显化

对应到 AIGC 与智能系统：

> 用户身份 + prompt 编写 + API 请求 + system 权限 + workflow 编排 + 模型执行 + 内容生成

由此可以得到赛博修仙的第一组根概念与运行公理。

> 灵气入府，炉阵转化为灵力；灵力驱动灵机；灵机以内在灵核运转符律，并在交互中显出灵识。

对应到现实技术：

> 电力进入数据中心，经 GPU 集群与调度系统转化为算力；算力驱动模型运行；模型以内在架构、权重与参数完成推理，并在交互中表现出认知能力。

| 根概念 | 技术映射 | 基本解释 |
| --- | --- | --- |
| 灵气 | 电力、能源、供能系统 | 数据中心与智能基础设施运行所需的原始能量 |
| 灵府 | 数据中心、AI 基础设施、云计算中心 | 灵气汇聚、灵力转化、灵机安置与运转之所 |
| 炉阵 | GPU 集群、计算集群、调度系统 | 将灵气转化为可用灵力的硬件与调度结构 |
| 灵力 | 算力、compute、可调度计算资源 | 灵机实际施展推理、生成、训练所消耗的能力 |
| 灵材 | 数据、语料、知识库、样本 | 被模型吸收、检索、炼化的原料 |
| 符 | token、可计算符号、语义片段 | 智能系统可识别和处理的最小符号单位 |
| 符纪 | context、上下文 | 当前交互中已经写入、累积并约束后续显化的符之脉络 |
| 咒 | prompt、指令序列、调用意图 | 驱动模型行为的语言结构 |
| 箓 | 权限、身份、access scope | 谁可以调用什么能力的授权体系 |
| 印 | 签名、认证、密钥、proof | 权威确认与身份凭证 |
| 坛 | runtime、session、容器、运行环境 | 法术发生的执行场 |
| 阵 | workflow、orchestration、protocol | 多步骤、多工具、多主体协作结构 |
| 灵机 | model、AI system、智能实体 | 能被调用、能响应、能推理、能生成的智能运行体 |
| 灵核 | model core、architecture、weights | 灵机内部运转的核心结构、能力根基与权重载体 |
| 灵枢 | API Gateway、系统接入中枢 | 诸法门与符门的统一接入、验印、授箓、路由与调度中枢 |
| 符律 | parameters、rules、model behavior law | 灵机运转所依凭的参数、规则与行为倾向 |
| 灵识 | apparent cognition、拟认知表征 | 灵机在交互中显出的理解、判断、记忆与一致性，不预设真实意识 |
| 显化 | generation、rendering、execution | 将潜在可能变为可见结果 |
| 天道 | protocol、consensus、system law | 约束世界运行的底层规则 |

## 四大主线

赛博修仙的操作体系可以分为四条主线。它们分别解释智能系统的使用、炼制、编排与基础设施。

| 主线 | 技术范围 | 核心问题 |
| --- | --- | --- |
| 符法 | prompt、inference、generation、交互 | 如何驱动灵机 |
| 丹法 | training、fine-tuning、调参、评测 | 如何炼成灵机 |
| 阵法 | workflow、agent、tool calling、多系统编排 | 如何组织多步行动 |
| 府阵 | data center、GPU cluster、cloud、network、deployment | 灵机如何获得栖身与运转之所 |

简言之：

> 符法解决“如何驱动灵机”，丹法解决“如何炼成灵机”，阵法解决“如何组织灵机行动”，府阵解决“灵力从何而来”。

## AIGC 映射

| 技术概念 | 赛博修仙命名 | 解释 |
| --- | --- | --- |
| Token | 符 | 最小可计算语义单位，万法由符起 |
| Context | 符纪 | 上下文，已经写入并持续影响后续生成的符之脉络 |
| Prompt | 符咒 | 由意图、约束、格式与上下文组成的调用指令 |
| System Prompt | 天条 / 本命咒 | 高优先级规则，决定灵机的根本行为边界 |
| Context Window | 符窗 | 一次施法可容纳的符量上限 |
| Inference | 驱符 / 起咒 | 调用已成模型，使其根据符咒显化结果 |
| Generation | 显化 | 文本、图像、音频、视频从潜在空间中成形 |
| Embedding | 符印 | 将语义压入可计算空间的印记 |
| Vector Database | 符库 | 存放符印并支持召回的库藏 |
| RAG | 搜神 / 查经 | 先检索外部知识，再辅助灵机作答 |
| Hallucination | 心魔 | 灵机生成看似合理但不可靠的幻象 |
| Alignment | 合道 | 让灵机行为符合人类价值、规则与边界 |
| Guardrail | 结界 | 阻挡危险输出或越界行为的安全边界 |
| Jailbreak | 破禁 | 绕过结界与天条的攻击性提示 |

## 灵机结构

模型不是单一概念。在赛博修仙体系里，模型至少可以拆成灵机、灵核、符律、灵智、灵识五层。

| 层级 | 技术映射 | 解释 |
| --- | --- | --- |
| 灵机 | model、AI system | 用户可调用的智能运行体，强调响应、推理、生成与行动能力 |
| 灵核 | model core、architecture、weights | 灵机内部的核心结构，决定能力边界、行为倾向与运行方式 |
| 符律 | parameters、rules、behavior patterns | 灵机运行所依的参数、规则、权重分布与行为法则 |
| 灵智 | intelligence、capability | 灵机表现出的解题、推理、归纳、生成等智能能力 |
| 灵识 | apparent cognition、拟主体性 | 灵机在交互中显出的理解、记忆、判断与自我一致性；它描述显现，不预设真实意识 |

一句话概括：

> 灵机是可被调用的智能体，灵核是其内在核心，符律是其运行法则，灵智是其能力，灵识是其在交互中显出的认知感。

## 丹法映射

“炼丹”是中文 AI 圈已经自然生成的工程黑话。本项目将其并入体系，作为模型训练、调参、实验与评测的核心分支。

| 技术概念 | 赛博修仙命名 | 解释 |
| --- | --- | --- |
| Data Collection | 采灵材 | 收集可供模型吸收的数据与语料 |
| Data Cleaning | 洗灵材 | 清洗、过滤、标注、去噪，使灵材可入炉 |
| Pre-training | 筑基 / 大炼 | 以大规模灵材奠定灵机根基 |
| Training | 炼化 / 修炼 | 模型吸收灵材、形成能力的过程 |
| Experiment Run | 开炉 | 启动一次训练、微调或调参实验 |
| Hyperparameter Tuning | 调火 / 控火 | 调学习率、batch size、训练轮次等火候 |
| Fine-tuning | 精炼 / 回炉 | 在既有根基上定向调整能力、风格或领域适配 |
| Loss Function | 劫数 / 丹劫 | 训练中必须持续面对并降低的误差关卡 |
| Checkpoint | 丹胚 / 丹成一转 | 某个训练阶段保存下来的中间成果 |
| Failed Run | 炸炉 | 训练崩坏、loss 爆炸、结果不可用 |
| Overfitting | 走火入魔 | 对训练集过度执着，泛化能力变差 |
| Underfitting | 根基未成 / 火候不足 | 模型尚未学到有效规律 |
| Model Distillation | 萃丹 / 传功 | 从大模型中提炼能力，炼成更小模型 |
| Evaluation | 试丹 / 试剑 | 检验模型效果、稳定性与可用性 |
| Benchmark | 试剑石 / 丹榜 | 用于比较模型成色的标准任务或榜单 |
| SOTA | 丹魁 / 榜首 | 当前公开评测中的最强结果 |

| --- | --- | --- |
| Electricity | 灵气 | 原始能源，供养灵府与炉阵 |

| Data Center | 灵府 | 灵气汇聚、灵力转化、灵机安置与运转之所 |

## API 与接入层映射

API 在本体系中不设单一母译名，而按能力类型直接分为法门与符门。这种二分本身就是中文语境对技术术语的再解释：确定性功能调用是一种法门，生成式灵机调用是一种符门。

| 技术概念 | 赛博修仙命名 | 解释 |
| --- | --- | --- |
| Traditional API | 法门 | 确定性功能调用门径，调用者按规约传入参数以使用某项能力 |
| LLM API | 符门 | 通向灵机的生成式调用门径，调用者递入符咒、符纪与参数，引动灵机显化 |
| API Gateway | 灵枢 | 诸法门与符门的统一接入中枢，负责验印、授箓、限流、路由、转发与调度 |
| API Request | 符令 | 一次传入门中的调用请求 |
| API Response | 回符 | 门内返回的结果或显化 |
| API Key | 法印 / 符钥 | 调用者进入法门或符门的身份凭证 |
| Authentication | 验印 | 验证调用者身份 |
| Authorization | 授箓 | 确认可调用的能力范围与权限 |

## 命名原则

一个好的赛博修仙术语，至少应该满足以下原则中的三项：

1. **技术准确**：不能为了好玩而误导技术机制。
2. **文化有根**：尽量来自中文语境中已有的意象、结构或词源。
3. **一听能懂**：非专业用户能凭直觉理解大概含义。
4. **可继续生长**：能衍生出相关词族，而不是孤立玩梗。
5. **适合使用**：能放进真实句子里，而不只是海报标题。
6. **边界清楚**：涉及宗教、民俗、神秘主义时，保持类比意识，不宣称真实效力。
